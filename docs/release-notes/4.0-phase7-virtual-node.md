# 4.0 — Phase 7: Per-Source Virtual Node

## Breaking change

Virtual Node configuration has moved from global environment variables to per-source settings. The following environment variables are **no longer honored** and have been removed entirely:

- `ENABLE_VIRTUAL_NODE`
- `VIRTUAL_NODE_PORT`
- `VIRTUAL_NODE_ALLOW_ADMIN_COMMANDS`

## Migration

If you previously ran MeshMonitor with `ENABLE_VIRTUAL_NODE=true`, you must re-enable Virtual Node on each source that should expose one after upgrading:

1. Open **Dashboard → Edit Source** (admin only) on the `meshtastic_tcp` source you want to expose.
2. In the **Virtual Node** section, enable the toggle.
3. Enter a port (the previous default was `4404`) and decide whether to allow admin commands.
4. Save.

Each source now has its own independent Virtual Node endpoint. Two sources on different nodes can each expose their own VN on different ports.

## Notes

- Virtual Node is currently only supported on `meshtastic_tcp` sources. MQTT and MeshCore sources ignore VN config.
- Ports must be unique across sources — the API rejects collisions with a 409.
- Changing VN config for an active source is hot-swapped without restarting the upstream TCP connection; changing host/port still triggers a full reconnect.
